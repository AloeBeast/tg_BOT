# tg_BOT — заметки для нейросети

Этот файл кратко описывает текущее устройство проекта, чтобы следующая нейросеть или разработчик быстрее поняли, что уже реализовано и где это лежит.

## Что это за проект

`tg_BOT` — Telegram-бот на Python с использованием `aiogram`. Бот отвечает пользователю в образе персонажа Виктора, хранит короткую историю диалога и простую карточку пользователя в SQLite, а также умеет принимать фотографии: сначала получает описание изображения через vision-модель, затем передаёт это описание основной текстовой модели для ответа в стиле персонажа.

## Точка входа и запуск

- Главная точка входа: `main.py`.
- При запуске создаётся объект `Bot`, `Dispatcher`, подключаются роутеры обработки фото и текста.
- Роутер фото подключён перед текстовым роутером, чтобы фото обрабатывались специализированным хендлером раньше универсального текстового обработчика.
- Перед стартом polling вызывается `init_db()`, который создаёт нужные таблицы SQLite, если их ещё нет.

Запуск ожидается примерно такой:

```bash
python main.py
```

Для запуска нужны переменные окружения, обычно через `.env`. Безопасный шаблон лежит в `.env.example`:

```env
BOT_TOKEN=telegram_bot_token
AI_API_KEY=api_key_for_openai_compatible_provider
AI_BASE_URL=https://example.com/v1
AI_MODEL=gemini-3.1-flash-lite
AI_MODEL_FAST=DeepSeek V4 Flash
AI_MODEL_VISION=gemini-3.1-flash-lite
DEBUG_MODE=false
HISTORY_LIMIT=8
```

Если одна из обязательных переменных `BOT_TOKEN`, `AI_API_KEY` или `AI_BASE_URL` отсутствует, `config.py` выбросит `ValueError` при импорте.

## Конфигурация моделей

Файл `config.py` загружает `.env` через `python-dotenv` и задаёт модели:

- `AI_MODEL` — основная модель для диалога с Виктором, по умолчанию `gemini-3.1-flash-lite`.
- `AI_MODEL_FAST` — модель для рутинных задач вроде извлечения имени и фактов; по умолчанию равна `AI_MODEL`, если не задана отдельно.
- `AI_MODEL_VISION` — модель для vision-запросов; по умолчанию равна `AI_MODEL`.
- `HISTORY_LIMIT` — единый лимит сообщений истории, по умолчанию `8`.
- `DEBUG_MODE` — флаг отладочного режима, по умолчанию выключен.

Клиенты OpenAI SDK создаются с `api_key=AI_API_KEY` и `base_url=AI_BASE_URL`, то есть проект рассчитан на OpenAI-compatible API, а не обязательно на официальный OpenAI endpoint.

## Персонаж бота

Образ бота описан в `persona.py` в константе `PERSONA_PROMPT`:

- имя персонажа — Виктор;
- стиль — грубый, саркастичный, короткий, с матом;
- ответы должны быть полезными и по делу;
- запрещены травля и дискриминационные оскорбления;
- при серьёзных темах вроде здоровья, утраты или тяжёлых переживаний бот временно отвечает без сарказма и хамства.

Этот промпт дополняется профилем пользователя в `services/ai_text.py` функцией `build_system_prompt()`.

## Работа с базой данных

База данных — SQLite-файл `bot_database.db`, имя задано в `database.py` как `DB_NAME`.

При инициализации создаются две таблицы:

### `messages`

Хранит историю сообщений:

- `id` — автоинкрементный первичный ключ;
- `user_id` — Telegram ID пользователя;
- `role` — роль сообщения (`user` или `assistant`);
- `content` — текст сообщения;
- `timestamp` — время создания, по умолчанию `CURRENT_TIMESTAMP`.

### `users`

Хранит простой профиль пользователя:

- `user_id` — Telegram ID пользователя, первичный ключ;
- `name` — имя пользователя, если оно было извлечено из сообщений;
- `facts` — список кратких фактов о пользователе текстом.

Основные функции:

- `init_db()` — создаёт таблицы;
- `save_message()` — сохраняет одно сообщение;
- `get_history()` — возвращает последние сообщения пользователя в хронологическом порядке;
- `get_user_profile()` — получает или создаёт профиль пользователя;
- `update_user_name()` — обновляет имя;
- `append_user_fact()` — добавляет факт и хранит только последние `max_facts`, по умолчанию 8.

## Текстовый сценарий

Файл: `handlers/text_chat.py`.

Что происходит при `/start`:

1. Бот отправляет короткое приветствие от имени Виктора.

Что происходит при обычном текстовом сообщении:

1. Проверяется наличие текста и `from_user`.
2. Отправляется Telegram chat action `typing`.
3. Сообщение анализируется функцией `extract_profile_update()` на имя и полезные факты о пользователе.
4. Если найдено имя — обновляется профиль.
5. Если найден факт — факт добавляется в профиль.
6. Сообщение пользователя сохраняется в `messages`.
7. Собирается системный промпт: персонаж + профиль пользователя.
8. Берутся последние 8 сообщений истории.
9. История отправляется в `get_ai_reply()`.
10. Ответ модели сохраняется и отправляется пользователю.

## Фото-сценарий

Файл: `handlers/photo_chat.py`.

Что происходит при фото:

1. Проверяется `from_user` и наличие фото.
2. Отправляется chat action `typing`.
3. Берётся фото в максимальном доступном качестве.
4. Файл скачивается через Telegram Bot API.
5. Байты фото отправляются в `describe_image()` из `services/ai_vision.py`.
6. Vision-модель возвращает подробное описание изображения.
7. Описание фото и подпись пользователя, если она есть, объединяются в текстовое сообщение.
8. Это объединённое сообщение сохраняется в историю как сообщение пользователя.
9. Основная модель получает системный промпт и историю, затем отвечает в стиле Виктора.
10. Ответ сохраняется и отправляется пользователю.

## AI-сервисы

### `services/ai_text.py`

Содержит:

- `get_ai_reply(messages)` — отправляет историю в chat completions и возвращает текст ответа. Ограничение `max_tokens=250`.
- `extract_profile_update(user_text)` — просит модель вернуть JSON вида `{"name": "", "fact": ""}`. Если модель вернула невалидный JSON или запрос упал, функция молча возвращает пустые поля.
- `build_system_prompt(profile)` — добавляет к `PERSONA_PROMPT` имя пользователя и известные факты.

### `services/ai_vision.py`

Содержит:

- `describe_image(image_bytes)` — кодирует изображение в base64, отправляет его как `data:image/jpeg;base64,...` в chat completions и просит подробно описать объекты, людей без идентификации личности, обстановку, текст и настроение фото. Ограничение `max_tokens=300`.

## Зависимости

Минимальные runtime-зависимости описаны в `requirements.txt`:

- `aiogram`;
- `openai`;
- `python-dotenv`.

`sqlite3`, `asyncio`, `base64`, `json`, `os` — из стандартной библиотеки Python.

## Текущие особенности и технический долг

- Нет тестов.
- Рабочая база `bot_database.db`, `.env`, временные файлы Python и `debug_photo.jpg` исключены через `.gitignore`.
- Зависимости описаны в `requirements.txt`.
- Безопасный шаблон обязательных переменных окружения описан в `.env.example`.
- Ошибки AI-запросов логируются через `logging`, а пользователю возвращается нейтральное сообщение без технических деталей.
- История ограничена единым значением `HISTORY_LIMIT`, по умолчанию 8 сообщений.

## Как безопасно продолжать разработку

1. Подумай над миграциями БД, если схема начнёт расширяться.
2. Добавь базовые тесты для `database.py` и функций сборки промпта.
3. При развитии фото-сценария рассмотри сжатие изображений и rate limit.
4. Не меняй стиль Виктора без явного запроса: это центральная задумка проекта.
